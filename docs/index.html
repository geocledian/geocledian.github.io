Changes from API v3 to API v4

Overview

  • modern tech stack with derived documentation from code, OpenAPI standard
  • minor versioning in root of API e.g. 4.0.1 on each release
  • new endpoint "colormaps"
  • reworked error handling:
    • content object does only carry valid data; in case of an error it is not present!
    • error object with error group name (e.g. "Authentication Error")
    • error detail object with detail message (e.g. "key validity expired") or list of errors in case of Client data error (HTTP 422)
    • Client data validation errors (HTTP 422 Unprocessable Entity) will be more verbose in response in field detail (per field)
    • HTTP status according to the error (e.g. HTTP 401 Unauthorized, HTTP 422 Unprocessable Entity)
